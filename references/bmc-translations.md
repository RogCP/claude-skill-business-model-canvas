## Usage Notes

### Language Detection

The skill should automatically detect the language of the conversation and use the appropriate translations from this table.

**Detection method:**
1. Analyze the user's messages for language patterns
2. If ambiguous, default to English
3. User can explicitly request a specific language

### Fallback Languages

If a language is not in this list, default to English and inform the user that custom translations are being used.

### Additional Supported Languages

The book has also been officially translated to:
- **Italian** (Italiano)
- **Korean** (한국어)
- **Polish** (Polski)
- **Dutch** (Nederlands)

For these languages, you may need to research the official translations from the respective editions of the book.

## Implementation in Skill

When generating a BMC, the skill should:

1. **Detect conversation language** from user messages
2. **Look up translations** in this reference table
3. **Apply translations** to:
   - Block titles in the HTML
   - Modal content (questions/descriptions)
   - Subtitle text
4. **Keep emojis consistent** across all languages
5. **Maintain professional terminology** from official translations

This ensures consistency with the global Business Model Canvas standard and provides users with familiar terminology from the Osterwalder framework.
